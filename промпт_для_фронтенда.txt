Напиши фронтенд для мессенджера на react.
Flow:
    1. Аутентификация или регистрация (логин, пароль)
    2. Подключение к websocket при успешной аутентификации
    3. Основное окно мессенджера
Там будет выбор светлой, темной темы (по умолчанию темная). Поддержка всех типов экранов (телефон, пк). Красивый и современный интерфейс.
При отправке сообщения, пока оно не подтверждено (пришел ивент на получении собственного сообщения) показывается иконка времени. Если сообщение прочитано - показывается 2 галочки, если отправлено (подтверждено) - одна галочка.
При создании чата, в нем нельзя писать, если оно не подтверждено.
При показе непрочитанного сообщения, нужно отправлять opcode для маркирования прочитанности. Свои отправленные сообщения дополнительно маркировать не нужно, они и так считаються прочитанными.
UI:
1. вход - страница входа
2. Основной интерфейс:
    1. Для пк - слева чаты, слайдер для изменения размера списков чатов, справа окно сообщений в чате. Список чатов - Название, фото (если нет, то отобразить аватар по умолчанию с заглавной буквой названия чата на фоне), текст последнего сообщения с ограничением по числу символов. Слева выше чатов 3 полоски, кликнув на них появляется профиль пользователя с информацией с его именем пользователя и логином для входа.
    2. Для телефона - список чатов на весь экран. Слева выше чатов 3 полоски, кликнув на них появляется профиль пользователя с информацией с его именем пользователя и логином для входа. При клике на чат появляется список сообщений.

Для действия с пользователями нужно использовать их только их id.

HTTP:
HTTP host: "localhost:1337"

Регистрация
POST /api/auth/signup

Принимает JSON
{
    "login": "",
    "username": "",
    "password": ""
}

При успехе:
    200 OK
При ошибке:
    400
    JSON body
    {
        "message": "описание ошибки"
    }

При ошибке:
    401 - не верный логин или пароль, 400 - остальные ошибки

Аутентификация
POST /api/auth/signin

Принимает JSON
{
    "login": "",
    "password": ""
}

Возвращает:
    При успехе:
        200 OK
        JSON body
        {
            "user": {"username": "", "id": 0},
            "token": "JWT token"
        }

Загрузка файла
POST /api/files

Принимает одиночный файл в MIME формате multipart

Возвращает:
    При успехе:
        200 OK
        JSON body
        {"fileId":"UUID str"}

    При ошибке:
        400
        JSON body
        {"message":"описание ошибки"}

Получение файла
GET /api/files

Принимает JSON
{
    "fileId": ""
    "chatId": ""
    "messageId": 0
}

Возвращает:
    При успехе:
        Headers:
            Content-Type: {mime type}
            Content-Length: {content len}
        body:
            содержимое файла в байтах

WEBSOCKET
URL -> localhost:1337/ws

requestId - id запроса.
В ответе сервер передает его обратно, что бы клиент смог сопоставить запрос и ответ.
Все запросы и ответы передаются через JSON.

Все сообщения от сервера имеют поле type, указывающий тип сообщения

Существуют типы сообщений:
    request:
        Отправляет клиент для совершения действия (отправка сообщения, создание чата, и тд)
        Включает в себя поле requestId.
        Если это действие подразумевает рассылку события всем пользователям в чате,
        дополнительно указывается userEventId. Далее этот идентификатор указывается в событии

        Запрос включает в себя:
            1. int requestId - id запроса
            2. int opcode - тип операций
            3. JSON payload - полезная нагрузка о действии

        Пример запроса:
            {
                "opcode": 1,
                "requestId": 1,
                "payload": {
                    ...
                }
            }

    response:
        Отправляет сервер в ответ на запрос пользователя
        Включает в себя поле requestId, полученный от запроса

        Ответ включает в себя:
            1. int requestId - id запроса
            2. type: "RESPONSE"
            3. JSON payload - полезная нагрузка о действии

        Пример ответа:
            {
                "requestId": 1,
                "type": "RESPONSE",
                "payload": {
                    "success": true/false,
                    при ошибке добавляется её описание:
                        "message": ""
                    ...
                }
            }
    event:
        Событие о конкретном действии.

        Событие включает в себя:
            1. type: "EVENT"
            2. int userEventId - id события, полученный от пользователя, породивший данное событие.
                Это нужно, что бы клиент мог сопоставить свой запрос с этим событием,
                что бы получить от него какие-то данные.
                Например:
                    Клиент отправил сообщение и указал там userEventId, сервер отвечает статусом ok.
                    Далее все клиенты в чате (все websocket сессии) получают событие о
                    приходе сообщения. Клиент, который породил это сообщение получает его же, но через событие
                    и помечает его как отправленный, получает из него messageId и прочее
            3. int opcode - тип события (пришло сообщение, чат создан и тд)
            4. JSON payload - полезная нагрузка о действии

        Пример события:
            {
                "type": "EVENT",
                "userEventId": int,
                "opcode": int,
                "payload": {
                    ...
                }
            }


Далее список кодов операций (действий), принимает и возвращает payload

Чаты:
    opcode 1:
        создание чата с пользователем 1 на 1
        принимает:
            {
                "userId": long - с кем создать чат,
                "userEventId": int
            }

        возвращает при успехе:
            {
                "success": true
            }

        возвращает при ошибке: (чат уже создан или такого пользователя нет)
            {
                "success": false,
                "message": String - описание ошибки
            }
    opcode 2:
        создание группового чата
        принимает:
            {
                "usersIds": List<long> - с кем создать чат,
                "name": String - название чата,
                "description": String,
                "photoId": UUID | null,
                "userEventId": int
            }

        возвращает при успехе:
            {
                "success": true
            }

        возвращает при ошибке:
            {
                "success": false,
                "message": String - описание ошибки
            }
    opcode 3:
        получение списка чатов
        принимает:
            null

        возвращает:
            [
                {
                    "chatId": String,
                    "participants": [
                        {
                            "id": long,
                            "username": String
                        }
                    ],
                    "lastMessageNumber": long,
                    "name": String | null,
                    "description": String | null,
                    "photoId": String | null,
                    "lastMessage": {
                        "senderId": long,
                        "messageText": String,
                        "sendingTime": long - unix timestamp in seconds
                    },
                    "unreadMessages": long
                },
                ...
            ]

    opcode 4:
        отправка сообщения.
        принимает:
            {
                "chatId": String,
                "messageText": String,
                "attachments": ["UUID", ...] - UUID вложений,
                "userEventId": int
            }

        возвращает при успехе:
            {
                "success": true
            }

        возвращает при ошибке:
            {
                "success": false,
                "message": String - описание ошибки
            }
    opcode 5:
        получение сообщений
        принимает:
            {
                "chatId": String,
                "start": long - начало диапазона,
                "limit": long
            }

        возвращает:
            {
                [
                    {
                        "messageNumber": long,
                        "senderId": long,
                        "chatId": String,
                        "usersViews": List<long>,
                        "messageText": String | null,
                        "attachments": List<String> - UUID вложений | null,
                        "sendingTime": long - unix timestamp in seconds
                    }
                ],
                ...
            }
    opcode 6:
        получение id по имени пользователя
        принимает:
            {
                "username": String
            }

        возвращает:
            {
                "users": [
                    {
                        "id": long,
                        "username": String
                    }
                ]
            }
    opcode 7:
        Сообщение просмотрено
        принимает:
            {
                "messagesNumbers": List<long>, - номера сообщений, которые прочитаны клиентом
                "chatId": String,
                "userEventId": int
            }

События:
    opcode 1:
        приватный чат был создан
        отправляется всем участникам:
            {
                "chatId": String,
                "users": [
                    {
                        "id": long,
                        "username": String
                    }
                ]
            }
    opcode 2:
        Групповой чат был создан
        Отправляется всем участникам:
            {
                "chatId": String,
                "participants": [
                    {
                        "id": long,
                        "username": String
                    }
                ],
                "name": String,
                "description": String
            }
    opcode 3:
        сообщение получено
        отправляется всем участникам:
            {
                "senderId": long,
                "chatId": String,
                "messageText": String | null,
                "attachments": List<String> - UUID вложений | null,
                "sendingTime": long - unix timestamp in seconds,
                "messageNumber": long
            }
    opcode 4:
        сообщения были прочитаны
        отправляется всем участникам:
            {
                "chatId": String,
                "messagesIds": List<long>,
                "userId": long - кто прочитал,
                "timestamp": long - когда прочитал. Время UTC +0
            }
