Domain правила:
У пользователя есть логин для входа в систему. Он никому не показывается. Нужен только для входа.
Он должен быть от 4 симлв без пробелов

Есть имя пользователя. Оно от 3 симвл.

Пароль от 8 симлв и должен содержать минимум по 1 - заглавных, строчных и цифр.

Для действия с пользователями нужно использовать их только их id.

Коммуникация с пользователем:
HTTP:
HTTP host: "localhost:1337"

Регистрация
POST /api/auth/signup

Принимает JSON
{
    "login": "",
    "username": "",
    "password": ""
}

При успехе:
    200 OK
При ошибке:
    400
    JSON body
    {
        "message": "описание ошибки"
    }

При ошибке:
    401 - не верный логин или пароль, 400 - остальные ошибки

Аутентификация
POST /api/auth/signin

Принимает JSON
{
    "login": "",
    "password": ""
}

Возвращает:
    При успехе:
        200 OK
        JSON body
        {
            "user": {"username": "", "id": 0},
            "token": "JWT token"
        }

Загрузка файла
POST /api/files

Принимает одиночный файл в MIME формате multipart

Возвращает:
    При успехе:
        200 OK
        JSON body
        {"fileId":"UUID str"}

    При ошибке:
        400
        JSON body
        {"message":"описание ошибки"}

Получение файла
GET /api/files

Принимает JSON
{
    "fileId": ""
    "chatId": ""
    "messageId": 0
}

Возвращает:
    При успехе:
        Headers:
            Content-Type: {mime type}
            Content-Length: {content len}
        body:
            содержимое файла в байтах

WEBSOCKET
URL -> localhost:1337/ws

requestId - id запроса.
В ответе сервер передает его обратно, что бы клиент смог сопоставить запрос и ответ.
Все запросы и ответы передаются через JSON.

Все сообщения от сервера имеют поле type, указывающий тип сообщения

Существуют типы сообщений:
    request:
        Отправляет клиент для совершения действия (отправка сообщения, создание чата, и тд)
        Включает в себя поле requestId.
        Если это действие подразумевает рассылку события всем пользователям в чате,
        дополнительно указывается userEventId. Далее этот идентификатор указывается в событии

        Запрос включает в себя:
            1. int requestId - id запроса
            2. int opcode - тип операций
            3. JSON payload - полезная нагрузка о действии

        Пример запроса:
            {
                "opcode": 1,
                "requestId": 1,
                "payload": {
                    ...
                }
            }

    response:
        Отправляет сервер в ответ на запрос пользователя
        Включает в себя поле requestId, полученный от запроса

        Ответ включает в себя:
            1. int requestId - id запроса
            2. type: "RESPONSE"
            3. JSON payload - полезная нагрузка о действии

        Пример ответа:
            {
                "requestId": 1,
                "type": "RESPONSE",
                "payload": {
                    "success": true/false,
                    при ошибке добавляется её описание:
                        "message": ""
                    ...
                }
            }
    event:
        Событие о конкретном действии.

        Событие включает в себя:
            1. type: "EVENT"
            2. int userEventId - id события, полученный от пользователя, породивший данное событие.
                Это нужно, что бы клиент мог сопоставить свой запрос с этим событием,
                что бы получить от него какие-то данные.
                Например:
                    Клиент отправил сообщение и указал там userEventId, сервер отвечает статусом ok.
                    Далее все клиенты в чате (все websocket сессии) получают событие о
                    приходе сообщения. Клиент, который породил это сообщение получает его же, но через событие
                    и помечает его как отправленный, получает из него messageId и прочее
            3. int opcode - тип события (пришло сообщение, чат создан и тд)
            4. JSON payload - полезная нагрузка о действии

        Пример события:
            {
                "type": "EVENT",
                "userEventId": int,
                "opcode": int,
                "payload": {
                    ...
                }
            }


Далее список кодов операций (действий), принимает и возвращает payload

Чаты:
    opcode 1:
        создание чата с пользователем 1 на 1
        принимает:
            {
                "userId": long - с кем создать чат,
                "userEventId": int
            }

        возвращает при успехе:
            {
                "success": true
            }

        возвращает при ошибке: (чат уже создан или такого пользователя нет)
            {
                "success": false,
                "message": String - описание ошибки
            }
    opcode 2:
        создание группового чата
        принимает:
            {
                "usersIds": List<long> - с кем создать чат,
                "name": String - название чата,
                "description": String,
                "photoId": UUID | null,
                "userEventId": int
            }

        возвращает при успехе:
            {
                "success": true
            }

        возвращает при ошибке:
            {
                "success": false,
                "message": String - описание ошибки
            }
    opcode 3:
        получение списка чатов
        принимает:
            null

        возвращает:
            [
                {
                    "chatId": String,
                    "participants": [
                        {
                            "id": long,
                            "username": String
                        }
                    ],
                    "type": "PERSONAL" | "GROUP"
                    "lastMessageNumber": long,
                    "name": String | null,
                    "description": String | null,
                    "photoId": String | null,
                    "lastMessage": {
                        "senderId": long,
                        "messageText": String,
                        "sendingTime": long - unix timestamp in seconds
                    },
                    "unreadMessages": long
                },
                ...
            ]

    opcode 4:
        отправка сообщения.
        принимает:
            {
                "chatId": String,
                "messageText": String,
                "attachments": ["UUID", ...] - UUID вложений,
                "userEventId": int
            }

        возвращает при успехе:
            {
                "success": true
            }

        возвращает при ошибке:
            {
                "success": false,
                "message": String - описание ошибки
            }
    opcode 5:
        получение сообщений
        принимает:
            {
                "chatId": String,
                "start": long - начало диапазона,
                "limit": long
            }

        возвращает:
            {
                [
                    {
                        "messageNumber": long,
                        "senderId": long,
                        "chatId": String,
                        "usersViews": List<long>,
                        "messageText": String | null,
                        "attachments": List<String> - UUID вложений | null,
                        "sendingTime": long - unix timestamp in seconds
                    }
                ],
                ...
            }
    opcode 6:
        получение id по имени пользователя
        принимает:
            {
                "username": String
            }

        возвращает:
            {
                "users": [
                    {
                        "id": long,
                        "username": String
                    }
                ]
            }
    opcode 7:
        Сообщение просмотрено
        принимает:
            {
                "messagesNumbers": List<long>, - номера сообщений, которые прочитаны клиентом
                "chatId": String,
                "userEventId": int
            }
    opcode 8:
        Редактирование сообщения
        Принимает:
            {
                "chatId": String,
                "messageText": String,
                "attachments": ["UUID", ...] - UUID вложений,
                "userEventId": int
            }

События:
    opcode 1:
        приватный чат был создан
        отправляется всем участникам:
            {
                "chatId": String,
                "users": [
                    {
                        "id": long,
                        "username": String
                    }
                ]
            }
    opcode 2:
        Групповой чат был создан
        Отправляется всем участникам:
            {
                "chatId": String,
                "participants": [
                    {
                        "id": long,
                        "username": String
                    }
                ],
                "name": String,
                "description": String
            }
    opcode 3:
        сообщение получено
        отправляется всем участникам:
            {
                "senderId": long,
                "chatId": String,
                "messageText": String | null,
                "attachments": List<String> - UUID вложений | null,
                "sendingTime": long - unix timestamp in seconds,
                "messageNumber": long
            }
    opcode 4:
        сообщения были прочитаны
        отправляется всем участникам:
            {
                "chatId": String,
                "messagesIds": List<long>,
                "userId": long - кто прочитал,
                "timestamp": long - когда прочитал. Время UTC +0
            }
